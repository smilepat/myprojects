# claude.ai/design 입력 지침 — 단일 슬라이드: "역량 ↔ 문항 매핑"

> 이 문서 전체를 복사해서 claude.ai/design에 붙여넣으세요.
> 슬라이드 1장(16:9)을 생성합니다.

---

## 0. 슬라이드 목적

한국 EFL 교육의 핵심 통찰 — **"한 문항은 여러 역량의 결합이다"** — 를 비전문가가 5초 안에 이해하게 만드는 양분 그래프(bipartite) 다이어그램이 중심인 프레젠테이션 슬라이드.

청중: 영어 콘텐츠 출판사 임원, 학교장, 학부모 등 비측정학 전공자.
사용 맥락: Apple 생태계 협업 컨소시엄 결성을 위한 18분 발표의 2번째 슬라이드.

## §B. 슬라이드 내용 (상→하 순서)

### 1. 상단 영역 (약 25% 높이)

```
[H2 작게]  역량 ↔ 문항
[H1 크게]  같은 문제, 다른 [역량]의 결합.
```

- "역량" 단어만 빨강(`--red`) 굵게 강조
- 나머지 텍스트는 흰색
- 좌측 정렬

### 2. 중앙 — 다이어그램 영역 (약 60% 높이)

흰 카드(`--card-bg`) 위, 라운드 코너 8px, 패딩 12px.
가로 폭은 슬라이드의 약 88% (max 1000px).

다이어그램 사양은 아래 §C에서 상세히 기술.

### 3. 하단 캡션 (약 15% 높이)

회색(`--muted`) 한 줄:

```
⭐ '같은 말 다르게 알아보기'는 3문항 모두에 필요한 핵심 역량 — 한 역량 보강이 다수 문항의 성취도를 동시에 끌어올린다.
```

## §C. 다이어그램 상세 사양 — 양분 그래프 (Bipartite)

### 캔버스
- 1000 × 600 (가로×세로 단위)
- 흰 배경

### 헤더 (다이어그램 내부 상단)

| 위치 | 텍스트 | 스타일 |
|------|------|------|
| 중앙 상단 (y=32) | "한 문항을 푸는 데 필요한 역량들" | 18px weight 700, `--card-fg` |
| 좌측 상단 (y=58) | "수능 문항 (예시 3개)" | 13px weight 600, `#64748b` |
| 우측 상단 (y=58) | "필요한 역량 (7개)" | 13px weight 600, `#64748b` |

### 좌측 — 수능 문항 카드 3개

세로로 배치, 각 카드 **260×140**, 라운드 코너 10px.
위치: x=20, y= 75 / 245 / 415 (간격 30px).

각 카드 구조:
- **상단 헤더 바** (높이 32px): `--card-fg` 다크 네이비 배경, 흰 글씨, 둥근 상단 코너만
- **본문 영역** (높이 108px): 흰 배경, 카드 테두리 1.5px `--card-fg`

| # | 헤더 | 본문 (2줄) | 인용 (회색 11px) |
|---|------|----------|------|
| A | **A. 주제파악** | 글 전체의 핵심 메시지를<br>한 문장으로 추리기 | 「다음 글의 주제로 적절한 것은?」 |
| B | **B. 빈칸추론** | 지문 속 빠진 표현을<br>맥락으로 채우기 | 「빈칸에 들어갈 말로 적절한 것은?」 |
| C | **C. 글의 순서** | 주어진 글 뒤로 이어질<br>문장의 순서 맞추기 | 「주어진 글에 이어질 순서는?」 |

### 우측 — 역량 노드 7개

세로로 배치, 각 노드 **230×42**, 알약 모양 (라운드 코너 21px).
위치: x=750, y= 47 / 122 / 197 / 272 / 347 / 422 / 497 (간격 33px).

| # | 텍스트 | 스타일 |
|---|------|------|
| 1 | 1. 단어 뜻 알기 | 일반 |
| 2 | 2. 문장 뼈대 잡기 | 일반 |
| **3 ⭐** | **★ 3. 같은 말 다르게 알아보기** | **강조** |
| 4 | 4. 앞뒤 잇기 (인과·대조) | 일반 |
| 5 | 5. 글의 줄기 보기 | 일반 |
| 6 | 6. 빠진 곳 채우기 | 일반 |
| 7 | 7. 흐름 맞추기 | 일반 |

**일반 노드 스타일**:
- 배경 `--card-light` (`#F1F5F9`)
- 테두리 1px `--card-gray` (`#94a3b8`)
- 텍스트 `--card-fg` 14px weight 500

**강조 노드 (3번 "같은 말 다르게 알아보기") 스타일**:
- 배경 `--red-soft` (`#FFE8E6`)
- 테두리 2px `--red` (`#FF3B30`)
- 텍스트 `--red` weight 700
- 좌측에 ★ 표시

### 연결선 — 카드 우측 중점 → 노드 좌측 중점 (총 15개)

각 선은 부드러운 베지어 곡선 (cubic). 시작점·끝점·강도는 다음 매핑표대로:

| 카드 | → 노드 | 강도 | 선 스타일 |
|------|------|------|----------|
| **A. 주제파악** | → 1. 단어 뜻 | 기본 | 회색 점선 1px, opacity 0.5 |
| A | → 2. 문장 뼈대 | 기본 | 회색 점선 1px, opacity 0.5 |
| A | → ★3. 같은 말 | **핵심** | 빨강 3.5px opacity 0.85 |
| A | → 5. 글의 줄기 | **핵심** | 빨강 3.5px opacity 0.85 |
| **B. 빈칸추론** | → 1. 단어 뜻 | 기본 | 회색 점선 1px |
| B | → 2. 문장 뼈대 | 기본 | 회색 점선 1px |
| B | → ★3. 같은 말 | **핵심** | 빨강 3.5px |
| B | → 4. 앞뒤 잇기 | **핵심** | 빨강 3.5px |
| B | → 5. 글의 줄기 | 보조 | 파랑 2.5px opacity 0.7 |
| B | → 6. 빠진 곳 | **핵심** | 빨강 3.5px |
| **C. 글의 순서** | → 1. 단어 뜻 | 기본 | 회색 점선 1px |
| C | → 2. 문장 뼈대 | 기본 | 회색 점선 1px |
| C | → ★3. 같은 말 | 보조 | 파랑 2.5px |
| C | → 4. 앞뒤 잇기 | **핵심** | 빨강 3.5px |
| C | → 7. 흐름 맞추기 | **핵심** | 빨강 3.5px |

**중요 시각 원칙**:
- 빨강 선이 가장 굵고 또렷해서 핵심 흐름이 즉시 보여야 함
- 점선이 한 곳에 몰리지 않고 카드별로 자연스럽게 발산
- 곡선의 control point는 시작점·끝점에서 가로로 200 단위 (즉 `M x1,y1 C x1+200,y1 x2-200,y2 x2,y2`)

### 범례 (다이어그램 하단, y=580)

가로 한 줄, 중앙 정렬:

```
━━ 핵심 역량 (Primary)    ━ 보조 역량 (Secondary)    ⋯⋯ 기본 입력 처리
```

각 항목:
- 빨강 굵은 선 32px + " 핵심 역량 (Primary)"
- 파랑 중간 선 32px + " 보조 역량 (Secondary)"
- 회색 점선 32px + " 기본 입력 처리"

12px, `--card-fg`.

## §D. 시각 의도 (디자이너용 — AI에게 주는 추가 컨텍스트)

이 슬라이드의 단 하나의 메시지:

> **수능 문항은 단일 능력이 아니라 여러 역량의 결합이다.**
> 그래서 학습 처방의 단위는 "유형"이 아닌 "역량"이어야 한다.

다이어그램이 강조해야 할 시각적 패턴:

1. **양분 구조의 즉시 가독성** — 좌(문제) → 우(능력) 흐름이 5초 안에 인식되어야 함
2. **★ 노드의 두드러짐** — 모든 카드에서 빨강 굵은 선이 ★ 노드로 향하는 패턴이 한눈에 보임
3. **3단계 강도 차별화** — 빨강 두꺼움(중요), 파랑 중간(보조), 회색 점선(기본)의 시각 위계
4. **곡선의 우아함** — 직선/꺾인선 금지. 부드러운 베지어 곡선만

**피해야 할 것**:
- 선이 직각으로 꺾이거나 직선으로 카드를 가로질러 시각적 노이즈 생성
- 검정 슬라이드 배경이 흰 다이어그램 영역으로 새는 것 (테두리·패딩 확실히)
- 영문 폰트로 한글이 렌더되어 자간 어색 (Pretendard 우선 시도)
- 카드 헤더의 다크 네이비가 너무 칙칙해지는 것 (`#0f172a` 정확히)
